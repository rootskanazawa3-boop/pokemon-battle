## 第1回：AIで設計書作成（2時間）

### **目標**
- AIツール（ChatGPT、Cursor、GitHub Copilotなど）を使って設計書を作成する
- ゲームの仕様を明確にする

### **内容（90分実習 + 30分説明）**

#### **1. AIツールの紹介と設定（15分）**
- 使用するAIツールの紹介
  - ChatGPT / Claude
  - Cursor AI
  - GitHub Copilot
- ツールの設定方法

#### **2. AIにプロンプトを送る（20分）**
- 設計書作成のプロンプト例：
  ```
  ポケモンバトルゲームの設計書を作成してください。
  要件：
  - React + TypeScriptで作成
  - 1対1のシンプルなバトル
  - コマンド選択（たたかう/モンスター/バッグ/にげる）
  - 技選択（最大4つ）
  - HP管理と勝敗判定
  
  含めるべき内容：
  - データ構造（モンスター、技、バトル状態）
  - UI構成（コンポーネント設計）
  - バトルロジックの流れ
  - ダメージ計算式
  ```

#### **3. 設計書の確認と修正（40分）**
- AIが生成した設計書を読む
- 不十分な点を指摘して修正
- プロンプトを改善して再度生成

**プロンプト改善例**：
```
前回の設計書をより詳しくしてください。
特に以下を追加：
- 各コンポーネントのprops定義
- 状態管理の詳細
- 具体的なコード例
```

#### **4. 設計書の完成（15分）**
- 最終的な設計書を確定
- 次回までの準備物を確認

### **成果物**
- 完成した設計書（Markdownファイル）
- AIとのやり取りの記録

### **学ぶポイント**
- AIを活用した設計の進め方
- プロンプトの書き方
- 設計書の重要性

## 第2回：AIで環境構築（2時間）

### **目標**
- AIの助けを借りて開発環境を構築する
- プロジェクトの初期設定を完了する

### **内容（90分実習 + 30分説明）**

#### **1. プロジェクト作成のAIプロンプト（20分）**
- AIに環境構築を依頼：
  ```
  以下の技術スタックでポケモンバトルゲームの
  プロジェクトを作成してください：
  
  - Vite + React + TypeScript
  - Tailwind CSS
  - 開発サーバーはポート3000
  
  必要なファイル：
  - package.json（依存関係含む）
  - vite.config.ts
  - tsconfig.json
  - tailwind.config.js
  - 基本的なディレクトリ構造
  ```

#### **2. 生成されたコマンドを実行（30分）**
- AIが提示したコマンドを実行
  ```bash
  npm create vite@latest pokemon-battle -- --template react-ts
  cd pokemon-battle
  npm install
  npm install -D tailwindcss postcss autoprefixer
  npx tailwindcss init -p
  ```
- エラーが出たらAIに質問
- 解決方法を学ぶ

#### **3. 初期ファイルの確認（20分）**
- 生成されたファイル構造を確認
- 各ファイルの役割を理解
- 必要に応じてAIに質問

#### **4. 開発サーバーの起動確認（20分）**
- `npm run dev` で起動
- ブラウザで確認
- 動作確認

### **成果物**
- 動作する開発環境
- プロジェクトの基本構造

### **学ぶポイント**
- 環境構築の手順
- AIを活用した問題解決
- エラーへの対処法

---

## 第3回：ライブコーディング（2時間）

### **目標**
- 講師と一緒にコードを書く
- AIの補助を受けながら実装する

### **内容（100分実習 + 20分説明）**

#### **1. データ構造の実装（25分）**
- AIに質問しながら型定義を作成：
  ```
  ポケモンバトルゲームの型定義を作成してください：
  - Bokemon（モンスター）
  - Move（技）
  - BattleState（バトル状態）
  
  以下の情報を含めてください：
  - id, name, hp, maxHp, attack, defense
  - moves配列
  - 現在のゲーム状態
  ```

#### **2. UIコンポーネントの実装（30分）**
- コンポーネントを1つずつ作成
- 各コンポーネントの役割を説明
- AIにコードレビューを依頼：
  ```
  このコンポーネントのコードをレビューしてください。
  改善点があれば教えてください。
  ```

#### **3. バトルロジックの実装（30分）**
- ダメージ計算関数を実装
- AIに質問しながら完成：
  ```
  ポケモンのダメージ計算式を実装してください。
  以下の条件を含めて：
  - 攻撃力と防御力の比率
  - ランダム要素（0.85〜1.0倍）
  - 最低ダメージ1を保証
  ```

#### **4. 状態管理とイベント処理（15分）**
- useStateで状態管理
- イベントハンドラーの実装
- 動作確認

### **成果物**
- 動作するゲームの基本機能
- 各ステップのコード

### **学ぶポイント**
- 段階的な実装方法
- AIを活用したコーディング
- デバッグの考え方

### **ライブコーディングの進め方**
1. **講師が画面共有**
   - CursorなどのAI統合エディタを使用
   - AIの提案をリアルタイムで確認

2. **生徒も一緒に書く**
   - 同じコードを書いてもらう
   - つまづいたら質問OK

3. **AIに質問しながら**
   - わからないことがあればAIに質問
   - AIの答えを一緒に確認

### **目標**
- GitとGitHubの基礎を学ぶ
- 作成したゲームを公開する

### **内容（90分実習 + 30分説明）**

#### **1. Gitの基礎と初期化（25分）**
- AIにGitの使い方を質問：
  ```
  Gitでコードを管理する方法を教えてください。
  初めての人向けに簡単に説明してください。
  ```
- `.gitignore`の作成
- 初回コミット

#### **2. GitHubアカウント作成とリポジトリ作成（20分）**
- GitHubアカウントの作成
- AIに手順を確認しながらリポジトリ作成
- リモートリポジトリの設定

#### **3. コードのプッシュ（25分）**
- Personal Access Token（PAT）の作成
- コードのプッシュ
- AIにエラー対応を質問：
  ```
  GitHubにプッシュしようとしたら
  エラーが出ました。どうすればいいですか？
  [エラーメッセージを貼り付け]
  ```

#### **4. GitHub Pagesで公開（20分）**
- AIにデプロイ方法を質問：
  ```
  Viteで作ったReactアプリを
  GitHub Pagesで公開する方法を教えてください。
  GitHub Actionsを使った方法で。
  ```
- ワークフローファイルの作成
- GitHub Pagesの設定
- 公開URLの確認

### **成果物**
- 公開されたゲームのURL
- GitHubリポジトリ

### **学ぶポイント**
- GitとGitHubの使い方
- デプロイの流れ
- エラー解決の方法

## 🎯 授業全体のポイント

### **AI活用のメリット**
1. **効率化**: 環境構築や設定が早い
2. **学習**: AIの説明で理解が深まる
3. **エラー対応**: エラーをすぐに解決できる
4. **自信**: AIの助けがあれば不安が減る

### **AIの使い方**
- **良い使い方**:
  - わからないことを質問する
  - コードレビューを依頼する
  - エラー解決を依頼する
  - ベストプラクティスを聞く

- **避けるべき使い方**:
  - コードを丸ごとコピーして理解しない
  - AIに頼りすぎて自分で考えない

### **段階的な指導**
1. **最初は一緒に**: 講師と一緒にAIと対話
2. **慣れてきたら**: 自分でAIに質問
3. **最後は**: AIの助けを借りながら自分で完成
